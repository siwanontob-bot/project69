# เว็บไซต์ Dashboard ติดตามแผนงานโครงการ โรงเรียนครุศาสตร์

เว็บไซต์ระบบติดตามแผนงานโครงการสำหรับผู้บริหาร โรงเรียนครุศาสตร์ พัฒนาขึ้นโดยใช้เทคโนโลยีเว็บที่เป็นมาตรฐาน (HTML5, Vanilla CSS และ JavaScript (ES6+)) โดยไม่มีขั้นตอนการ Build ทำให้รองรับการเผยแพร่ผ่าน GitHub Pages ได้อย่างง่ายดาย

## คุณสมบัติหลัก (Key Features)

1. **การดึงข้อมูลแบบ Real-time**: ดึงข้อมูลสดใหม่โดยตรงจาก Google Sheet ทุกครั้งที่เปิดหรือโหลดหน้าเว็บใหม่
2. **สรุปผลสำหรับผู้บริหาร (Executive Summary)**: แสดงผลในรูปตัวเลขสรุป (KPIs) ที่สำคัญ เช่น จำนวนโครงการทั้งหมด, งบประมาณรวม, ยอดเบิกจ่ายสะสม, งบประมาณคงเหลือ และความคืบหน้าภาพรวม
3. **การนำเสนอด้วยกราฟ (Data Visualization)**:
   - กราฟวงกลม (Doughnut Chart) แสดงสัดส่วนสถานะโครงการ (ดำเนินการแล้ว, อยู่ระหว่างดำเนินการ, ยังไม่ดำเนินการ)
   - กราฟแท่ง (Bar Chart) เปรียบเทียบงบประมาณที่ได้รับการจัดสรรกับงบประมาณที่ใช้ไปจริง แยกตามแต่ละกลุ่มงาน
4. **ตารางรายละเอียดโครงการ**: แสดงรายชื่อโครงการ ผู้รับผิดชอบ งบประมาณ ความคืบหน้า (Progress bar) และสถานะ สามารถพิมพ์ค้นหาโครงการ และกรองข้อมูลตามกลุ่มงานหรือสถานะได้ทันที
5. **การแสดงผลที่เป็นมิตร (Responsive Design)**: รองรับการแสดงผลหน้าจอมือถือ แท็บเล็ต และหน้าจอคอมพิวเตอร์ขนาดใหญ่

## วิธีการรันในเครื่องส่วนตัว (Local Execution)

คุณสามารถเปิดใช้งานเว็บไซต์ในเครื่องคอมพิวเตอร์ของคุณเองได้สะดวกสบายและไม่มีปัญหา CORS:
1. **ดับเบิ้ลคลิกไฟล์ `index.html`** เพื่อเปิดใช้งานบนเว็บเบราว์เซอร์ได้ทันที! (ระบบรองรับการรันผ่านระบบไฟล์ `file://` โดยตรงแล้ว เนื่องจากใช้เทคนิคดึงข้อมูลแบบ JSONP จาก Google Visualization API ทำให้ไม่มีปัญหาการบล็อกความปลอดภัยของเบราว์เซอร์)
2. หรือจะใช้งานเซิร์ฟเวอร์จำลองเพื่อทดสอบการทำงานทั่วไป เช่น:
   - **VS Code**: ติดตั้งปลั๊กอิน `Live Server` แล้วกดคลิกขวาที่ไฟล์ `index.html` เลือก `Open with Live Server`
   - **Python**: รันคำสั่ง `python -m http.server 8000` ในโฟลเดอร์นี้ แล้วเปิดเบราว์เซอร์ที่ `http://localhost:8000`

## ขั้นตอนการนำเว็บไซต์ขึ้น GitHub เพื่อเผยแพร่ (GitHub Pages Deployment)

ทำตามขั้นตอนด้านล่างนี้เพื่อนำเว็บไซต์เผยแพร่สู่สาธารณะผ่านระบบ GitHub Pages (ฟรีและเสถียรมาก):

### ขั้นตอนที่ 1: เตรียมโค้ดของคุณด้วย Git
1. เปิดโปรแกรม Terminal, PowerShell หรือ Git Bash ในโฟลเดอร์นี้
2. เริ่มต้น Git Repository ด้วยคำสั่ง:
   ```bash
   git init
   ```
3. เพิ่มไฟล์ทั้งหมดเข้าสู่ระบบ Git:
   ```bash
   git add .
   ```
4. บันทึกประวัติการพัฒนา (Commit):
   ```bash
   git commit -m "Initial commit: School of Education Dashboard"
   ```
5. ตั้งชื่อกิ่งหลักเป็น `main`:
   ```bash
   git branch -M main
   ```

### ขั้นตอนที่ 2: สร้าง Repository บน GitHub
1. ลงชื่อเข้าใช้บัญชี GitHub ของคุณที่ [https://github.com](https://github.com)
2. คลิกปุ่ม **New** (หรือเครื่องหมายบวก `+` ที่มุมขวาบน) เพื่อสร้าง Repository ใหม่
3. กรอกรายละเอียด:
   - **Repository name**: เช่น `education-dashboard`
   - **Public/Private**: เลือกเป็น **Public** (จำเป็นสำหรับบริการ GitHub Pages ฟรี)
   - ไม่ต้องติ๊กเลือก Add a README, .gitignore หรือ license (เพราะเราสร้างไว้แล้วในเครื่อง)
4. คลิกปุ่ม **Create repository**

### ขั้นตอนที่ 3: อัปโหลดโค้ดขึ้น GitHub

คุณสามารถเลือกอัปโหลดไฟล์ด้วยวิธีใดวิธีหนึ่งดังนี้:

#### วิธีที่ 3.1: อัปโหลดผ่าน Command Line (สำหรับผู้มี Git บนเครื่อง)
รันคำสั่งเหล่านี้ใน Terminal ของคุณ:
```bash
git remote add origin https://github.com/ชื่อผู้ใช้ของคุณ/education-dashboard.git
git push -u origin main
```
*(แทนที่ `ชื่อผู้ใช้ของคุณ` ด้วยชื่อบัญชี GitHub จริงของคุณ)*

#### วิธีที่ 3.2: อัปโหลดผ่านหน้าเว็บไซต์ GitHub (วิธีที่ง่ายที่สุด ไม่ต้องใช้ Git CLI)
1. ในหน้า Repository ที่คุณเพิ่งสร้างขึ้น ให้มองหาข้อความ **"uploading an existing file"** (จะอยู่ในย่อหน้าแรกใต้หัวข้อ Quick setup)
2. ลากไฟล์จากในโฟลเดอร์นี้ทั้งหมด ได้แก่:
   - `index.html`
   - `styles.css`
   - `app.js`
   - `README.md`
   มาวางลงในช่องอัปโหลดบนหน้าจอเบราว์เซอร์
3. เลื่อนลงมาด้านล่าง กรอกข้อความสั้นๆ ในช่อง Commit changes เช่น `Initial commit` จากนั้นคลิกปุ่ม **Commit changes** เพื่อบันทึกไฟล์ขึ้นสู่ GitHub

---

### ขั้นตอนที่ 4: เปิดใช้งาน GitHub Pages เพื่อเผยแพร่เว็บ
1. ไปที่หน้า Repository ของคุณบนเว็บไซต์ GitHub
2. คลิกที่เมนู **Settings** (รูปฟันเฟืองที่แถบเครื่องมือด้านบน)
3. เมนูด้านซ้ายมองหาหมวดหมู่ **Code and automation** และคลิกเลือก **Pages**
4. ภายใต้หัวข้อ **Build and deployment**:
   - Source: เลือก **Deploy from a branch**
   - Branch: เลือกกิ่งเป็น `main` (หรือกิ่งที่มีไฟล์ของคุณ เช่น `main` หรือ `master`) และโฟลเดอร์เป็น `/ (root)`
5. คลิกปุ่ม **Save**
6. รอประมาณ 1-2 นาที จากนั้นรีเฟรชหน้า Settings -> Pages คุณจะได้รับลิงก์ URL สีเขียวสำหรับเข้าชมหน้าเว็บ เช่น:
   `https://ชื่อผู้ใช้ของคุณ.github.io/education-dashboard/`

สามารถส่งต่อลิงก์นี้ให้ผู้บริหารและผู้ใช้คนอื่นๆ เข้ามาเปิดดูข้อมูลได้ทันทีจากทุกที่!
